# project_pixelpet V1 开发执行口径清单

## 1. 目标与边界（统一口径）
项目目标：
- 做出 `Android 原生 + Widget-first + 单人离线养成` 的可用首版。
- 验证 `桌面常驻率`、`日常回访频次`、`基础养成留存`。

硬边界：
- 不做悬浮窗宠物（不走 Overlay 路线）。
- 不做联机与社交共养。
- 不做复杂道具经济与小游戏系统。
- 不做多宠物系统。

体验原则：
- Widget 是主入口，App 详情页是信息承接。
- 真实状态由“时间推导 + 快照”计算，不依赖高频后台刷新。
- App 与 Widget 必须共用同一套状态逻辑与同一数据源。

## 2. 版本范围（P0 / P1）
### P0（必须完成）
- 首次领养闭环：`首次进入 -> 领养 -> 引导放置 Widget -> 第一次互动`。
- 单宠物四状态：`饱腹/心情/清洁/精力`，统一量纲（建议 `0-100`）。
- 四核心动作：`喂食/玩耍/清洁/睡觉`，动作后有即时反馈。
- Widget 两规格：`2x2`、`4x2` 可放置与可见。
- Widget 主交互可用：至少 1 个高频 CTA 可直达结算。
- App 详情页可看完整状态，且与 Widget 状态一致。
- 离线可玩：无网络不影响核心流程。

### P1（在 P0 稳定后）
- 成长记录页（阶段变化 + 关键互动摘要）。
- 轻提醒（饥饿/困倦/脏了/成长完成）。
- 背包分区（轻量道具，不引入复杂经济规则）。
- 基础设置（提醒开关、音效开关、显示偏好）。
- `4x2` 鱼缸 Widget 方案（颜文字鱼漂游与轻互动）作为差异化增强。

## 3. 模块职责（开发实现口径）
- `PetRepository`：状态快照、日志、库存、成长记录读写。
- `PetStateCalculator`：时间衰减、边界裁剪、动作增量、阶段判定。
- `PetActionUseCase`：动作统一入口，串联结算、持久化、刷新、日志。
- `WidgetUpdater`：刷新所有 Widget 实例，保证多实例一致。
- `ReminderScheduler`：通过 WorkManager 做低频检查与提醒。
- `AssetMapper`：状态到视觉映射（宠物帧/文案/CTA）。

约束：
- 任何入口（Widget/App/提醒）触发动作，都必须走 `PetActionUseCase`。
- 禁止在 Widget 层和 App 层各自维护独立状态副本。
- `4x2` 鱼缸若进入 V1，只做轻反馈，不演变成复杂小游戏。

## 4. 数据与状态口径
核心对象：
- `pet`：唯一主宠物当前可信状态。
- `pet_action_log`：关键动作日志，含动作来源与前后快照。
- `inventory_item`：轻量道具数量。
- `growth_record`：阶段与里程碑摘要。
- `user_settings`：DataStore 偏好配置。

状态结算口径：
- 读状态：先读取快照，再按 `now - last_updated_at` 推导当前状态。
- 写状态：动作结算后写回新快照与关键时间字段。
- 阈值：状态有上下限，避免越界。
- 一致性：Widget 与 App 都基于同一结算结果展示。

## 5. 页面与入口口径
页面最小集合：
- 启动初始化页
- 首次领养页
- Widget 放置引导
- 宠物详情主页
- 成长记录页
- 设置页

入口分工：
- Widget：高频查看 + 一步交互 + 快速反馈。
- App：完整状态、成长记录、设置与次级信息。

## 6. 验收清单（发布前必过）
- 首次流程可完整走通：`领养 -> 放置 Widget -> 第一次喂食`。
- `冷启动/过夜/重启设备` 后状态可恢复且合理。
- Widget 与详情页状态一致，无明显不同步。
- `2x2` 和 `4x2` 无裁切、文案不溢出。
- 点击 Widget CTA 后，用户在 1 秒内看到明确反馈。
- 15 分钟以上未打开 App 后，状态衰减合理。
- 离线环境下核心互动全部可完成。
- 后台与电量开销可控，不依赖精确闹钟。

## 7. 风险回归重点（每轮必测）
- 状态重复结算或漏结算（尤其短时间频繁点击）。
- 多个 Widget 实例刷新不一致。
- 不同桌面启动器下布局错位或交互异常。
- 小屏中文文案截断与可读性问题。
- 提醒误触发、重复触发或无法关闭。

## 8. 执行节奏建议（不含排期）
- 先做“状态可信源”再做“视觉丰富度”。
- 先保稳定，再加鱼缸互动特效。
- 每完成一个功能块，立刻走一次验收清单中对应项。

---
本清单作为 V1 开发执行基线，后续如新增需求，需先标注其归属：`P0`、`P1` 或 `V1 外`，再进入实现。
