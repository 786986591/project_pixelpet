# project_pixelpet 页面结构

## 文档目标
本页用于把 V1 从“产品方向”落到“页面承载关系”。

页面结构只覆盖当前版本必须存在的页面与入口，不展开高保真交互稿，也不提前为未来社交和商城设计复杂页面。

## 页面总览
V1 建议只保留 5 个核心页面层级：
- 启动与初始化
- 首次领养流程
- 宠物详情主页
- 成长记录页
- 设置页

背包在 V1 中不单独拆成复杂二级系统，可先作为宠物详情页中的一个分区或弹层承载。

## 页面结构图
`启动页 -> 首次领养页 -> Widget 放置引导 -> 宠物详情主页 -> 成长记录页 / 设置页`

桌面 Widget 是最重要的入口，但不算一个 App 页面：
- 负责高频、轻量、即时的交互
- 在需要更完整信息时跳转到宠物详情主页

## 页面说明
### 1. 启动与初始化
页面目标：
- 完成应用冷启动
- 判断是否已有宠物存档
- 决定进入首次领养流程还是直接进入宠物详情主页

核心内容：
- 品牌或宠物开场画面
- 本地数据初始化
- 首次用户分流逻辑

退出条件：
- 无存档：进入首次领养页
- 有存档：进入宠物详情主页

### 2. 首次领养页
页面目标：
- 完成 V1 的首次宠物创建
- 向用户建立“这是桌面宠物”而不是“全屏游戏”的心智

核心内容：
- 宠物初始形象展示
- 简短产品说明
- `开始领养` 主按钮
- 领养完成后的 Widget 放置引导

V1 约束：
- 不提供复杂命名、捏宠、种族选择
- 默认只创建 1 只主宠物

### 3. Widget 放置引导
页面目标：
- 引导用户完成最关键的一步：把 Widget 放到桌面
- 尽量提高 `桌面常驻率`

核心内容：
- 为什么建议放置 Widget
- 2x2 和 4x2 的差异说明
- 引导操作文案
- 放置完成后的返回动作

成功标准：
- 用户知道如何放置 Widget
- 用户放置后能马上进行第一次互动

### 4. 宠物详情主页
页面目标：
- 承接 Widget 之外的完整信息展示
- 作为 V1 的主页面和状态总入口

建议模块：
- 宠物主展示区
- 四项状态区：饱腹、心情、清洁、精力
- 快捷操作区：喂食、玩耍、清洁、睡觉
- 背包区
- 成长信息摘要区
- 设置入口

进入方式：
- 冷启动后直接进入
- 从 Widget 点击进入
- 从通知或提醒进入

页面原则：
- 一屏内能看懂宠物当前状况
- 主要操作不应超过一次点击
- 背包与成长信息可以次级展开，但主状态与主动作必须始终可见

### 5. 成长记录页
页面目标：
- 给用户一个“宠物确实在成长”的可追溯感
- 承接阶段变化、关键行为、关系积累

建议内容：
- 当前成长阶段
- 已经历的成长节点
- 关键互动记录摘要
- 未来阶段的占位说明

V1 约束：
- 只做轻量记录，不做复杂时间轴叙事
- 只展示和养成有关的核心记录

### 6. 设置页
页面目标：
- 承接用户偏好与提醒控制
- 保持产品轻量，不把设置做成复杂配置中心

建议内容：
- 提醒开关
- 音效开关
- 主题或显示偏好
- 关于产品与版本信息

V1 不做：
- 账号绑定
- 云同步
- 复杂通知分类

## Widget 与页面分工
### Widget 负责
- 快速看状态
- 快速做一次操作
- 快速收到反馈

### App 页面负责
- 看完整状态
- 查看背包与成长记录
- 处理设置与次级信息

## V1 页面边界
- 页面数量尽量少，优先保证核心回路清晰
- 详情主页承担信息中枢角色，避免早期拆出过多页面
- 若后续要做背包系统扩展，再考虑把背包从详情页中独立
