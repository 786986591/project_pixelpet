# project_pixelpet 美术与动画规范

## 美术方案总览
当前保留两条并行视觉方向：
- `方案 A`：像素复古宠物
- `方案 B`：`4x2` 鱼缸 Widget + 颜文字鱼

两条方案同时保留，当前不互相覆盖：
- 方案 A 继续作为主宠物养成的基础视觉方案
- 方案 B 作为更轻、更桌面装饰化、也更有 Widget 识别度的扩展方向

## 方案 A：像素复古宠物
V1 原始美术风格锁定为：`8-bit / 16-bit 像素复古，可爱但不幼稚`。

### 目标气质
- 有陪伴感，但不过度低幼
- 有复古游戏感，但不要做成纯怀旧模仿品
- 在桌面上长时间存在时，视觉上应足够耐看、简洁、辨识度高

### 视觉原则
- Widget 是首要视觉场景，信息密度要低，角色辨识度要高
- 角色演出以关键状态反馈为主，不依赖持续运动
- App 内视觉可以比 Widget 略丰富，但不能偏离同一套角色设定和像素语言
- 首版优先做“稳定统一”，而不是追求大量皮肤或复杂装饰

### 工具建议
- 角色设计：`Aseprite`
- UI 绘制：`Figma`
- 像素资源导出：`PNG sprite sheet + JSON frame map`

### 动画策略
#### Widget 动画
- 使用按状态切换的关键帧位图
- 避免在 Widget 中做复杂实时动画
- 将动画重点放在点击反馈、状态变化、成长节点

#### App 内动画
- 可使用轻量帧动画或循环精灵动画
- 动画应服务状态表达，不应掩盖信息层级

#### 音效策略
- 首版只需要极简电子音
- 优先支持 `点击`、`喂食`、`成长` 三类反馈音

### 方案 A 资源清单
- 1 套主宠物形象
- 4 个成长阶段：孵化、幼体、成长期、成熟体
- 每个阶段至少 5 个状态帧：待机、开心、饥饿、困倦、脏乱
- 4 类交互特效：食物、爱心、泡泡、`Zzz`
- 2 套 Widget 背景：浅色、深色
- 3 个食物道具：基础粮、零食、恢复剂

## 方案 B：4x2 鱼缸与颜文字鱼
### 核心概念
把 `4x2` Widget 设计成一个桌面小鱼缸，鱼缸内部不是传统像素鱼，而是由 `颜文字` 组成的“情绪鱼”。

例如：
- `(^_^)`
- `(o_o)`
- `>_<`
- `(-_-)`
- `(≧▽≦)`

这些“鱼”不是纯文字标签，而是具有鱼形轮廓和漂游动作感的视觉角色：
- 颜文字本身是角色脸
- 外部可搭配极简鱼身、尾鳍、气泡、波纹
- 保持一眼就能认出“这是颜文字鱼”，而不是普通文本贴片

### 视觉方向
整体风格建议为：`轻拟物鱼缸 + 数字表情角色 + 干净桌面装饰感`。

目标气质：
- 更轻松、幽默、解压
- 比传统宠物更抽象，也更适合长期桌面常驻
- 比纯像素方案更现代、更新奇，能形成强识别点

### 视觉原则
- `4x2` 主要展示“水体空间感”和“漂游感”，而不是堆信息
- 颜文字鱼必须首先像一个角色，其次才是一个状态标签
- 单个鱼缸里的角色数量要克制，避免变成杂乱弹幕
- 所有动效都应缓慢、轻柔、可长期观看，不做高频跳动

### 鱼缸构图建议
- 背景层：玻璃边框、水体渐变、微弱高光
- 中景层：颜文字鱼、气泡、小海草、装饰石子
- 前景层：少量漂浮颗粒或反光

构图重点：
- 中间 60% 区域留给鱼的漂游路径
- 左右两端适合做往返游动和转向
- 底部装饰不应挡住主体角色

### 颜文字鱼设计原则
- 每条鱼尽量使用短颜文字，控制在 `3-8` 个可见字符内
- 优先选轮廓清楚、情绪明确、缩小后仍能辨认的颜文字
- 通过不同颜文字表达心情、精力、饱腹等状态变化

推荐情绪映射示例：
- 开心：`(^_^)`
- 饥饿：`(o_o)`
- 困倦：`(-_-)`
- 紧张：`>_<`
- 被互动后兴奋：`(≧▽≦)`

### 鱼缸互动设计
用户在 `4x2` 鱼缸 Widget 上的交互建议以“逗鱼”而不是“按钮操作”为主。

可用互动：
- 点击鱼缸空白处：激起一圈水波
- 点击单条颜文字鱼：该鱼短暂加速、转向、吐泡泡或切换表情
- 喂食操作：从顶部落下小鱼食，引发鱼群聚集
- 长按或二次点击：跳转宠物详情页或鱼缸详情页

互动原则：
- 优先做“可爱反馈”，其次才是功能性操作
- 鱼的反应应有轻微随机性，避免机械一致
- 单次反馈应在 `1 秒内` 感知到

### 漂游与动效策略
- 颜文字鱼应沿缓慢曲线路径漂游，而不是直线匀速移动
- 漂游速度应偏慢，保持治愈感
- 停顿、转向、轻微上下浮动，比持续高速运动更重要
- 同屏最多 `1-3` 条主鱼，避免拥挤

动效建议：
- 微气泡上浮
- 水波纹扩散
- 鱼尾轻摆
- 颜文字轻微角度变化或位置浮动，模拟游动

### 色彩方向
推荐避开高饱和儿童玩具色，改用更干净的水族箱调性：
- 水体：浅青、湖蓝、玻璃白
- 点缀：珊瑚橙、海草绿、柠檬黄
- 夜间版本：深蓝灰底 + 柔和发光气泡

### 方案 B 资源清单
- `4x2` 鱼缸背景 2 套：日间 / 夜间
- 颜文字鱼 6-10 款基础形态
- 气泡、水波、鱼食、海草等小元素
- 鱼被点击、喂食、转向时的轻反馈动效

## 通用输出规范
### 文件组织建议
- 按 `方案 / 角色 / 状态 / 用途` 维度组织资源
- 方案 A 和方案 B 的 Widget 资源分目录，避免混用

### 命名建议
统一使用英文小写加下划线，便于程序引用，例如：
- `pet_egg_idle`
- `pet_child_hungry`
- `kaomoji_fish_happy_01`
- `tank_bg_day`

### 导出要求
- 角色与特效统一按像素整数倍或统一缩放基准导出
- 避免在导出后再次缩放造成模糊或字符边缘发虚
- 帧图与 `JSON frame map` 一一对应，便于程序映射

## Widget / App 视觉边界
### Widget
- 优先保证角色清晰与状态可读
- 文案尽量短，不依赖多行说明
- `2x2` 可以保留主宠物卡
- `4x2` 可以扩展为鱼缸主题互动空间

### App
- 用于承接更完整信息，如状态详情、背包、成长记录、设置
- 如果采用鱼缸方案，App 内也应有对应主题页或状态说明，避免 Widget 与 App 风格完全断裂

## 首版资源优先级
优先级从高到低建议如下：
1. 方案 A 主宠物四阶段基础帧
2. 方案 B `4x2` 鱼缸背景与颜文字鱼基础动效
3. Widget 状态反馈资源与点击反馈
4. 交互特效
5. App 内补充动画
6. 额外装饰与可替换皮肤

## 当前结论
- 方案 A 继续承担“宠物养成主视觉”的角色
- 方案 B 为产品增加了一个很有辨识度的 `4x2` Widget 方向：`鱼缸 + 颜文字鱼 + 漂游互动`
- 如果后续要做设计稿，最值得先出的就是方案 B 的 `4x2` 鱼缸布局和 3-5 条颜文字鱼状态草图
