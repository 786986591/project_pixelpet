# project_pixelpet 验收与测试

## 验收目标
V1 的测试重点不是覆盖所有未来可能性，而是确认当前产品方向是否成立、核心体验是否稳定。

## 核心验收标准
- 用户首次安装后，能顺利完成 `领养 -> 放置 Widget -> 第一次喂食`
- 宠物在 `冷启动`、`熄屏过夜`、`重启设备后` 状态仍能正确恢复
- Widget 与 App 内详情页展示的状态一致
- `2x2` 与 `4x2` 组件在主流 Android 桌面上不出现裁切与文字溢出
- 用户点击 Widget CTA 后，能在 `1 秒内` 看到明确反馈
- 15 分钟以上未打开 App，回到桌面时宠物状态会合理衰减
- 电量与后台占用不过重，不依赖精确闹钟
- 无网络时完整可玩

## 核心测试场景
### 日常养成场景
- 正常养成 1 天，验证状态变化、交互反馈、成长推进是否合理
- 多次往返桌面与 App，验证状态展示是否一致

### 长时间未操作场景
- 连续 12 小时未操作，验证状态衰减和恢复是否合理
- 熄屏过夜后再次查看，验证时间推导逻辑是否稳定

### 高频交互场景
- 连续点击 `喂食` 与 `玩耍`，验证边界值与冷却处理
- 在短时间内多次进入详情页与 Widget 交互，验证状态是否重复结算

### 多实例与主题场景
- 同时放置多个 Widget 实例，验证刷新一致性
- 切换浅色 / 深色主题，验证背景与文案可读性

### 文案与适配场景
- 中文文案在小屏设备上不截断、不溢出
- 不同桌面环境下，角色、按钮、文案都保持可读

## 边界条件
- 无网络
- App 长时间未打开
- 系统未按预期时间触发后台任务
- 用户连续快速操作
- 设备重启后恢复

## 风险回归重点
- 状态推导是否与上次快照一致衔接
- Widget 刷新是否会漏掉某些实例
- App 与 Widget 是否出现不同步展示
- 不同尺寸下是否因资源或布局差异导致视觉错位

## 兼容性关注点
- 主流 Android 桌面启动器的 Widget 尺寸适配差异
- 浅色 / 深色背景下的角色与文字对比度
- 小屏设备上的信息密度控制

## 发布前检查项
- 首次引导路径完整可走通
- 两种 Widget 规格都已验证
- 离线状态下所有核心交互可完成
- 核心提醒不会误触发或重复触发
- 日志、崩溃收集与调试信息满足首版排查需求

## 当前测试结论
- V1 是否可发布，最终取决于“状态推导正确性”和“桌面体验稳定性”
- 如果这两项不稳定，即使功能数量不多，也会直接伤害产品可信度
