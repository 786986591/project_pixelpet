# project_pixelpet PRD

## 产品定位
`project_pixelpet` V1 是一款 `Android 原生、桌面 Widget 优先、单人养成、离线可玩、像素复古风` 的轻量虚拟宠物产品。

它不是传统全屏宠物游戏，也不是悬浮桌宠工具，而是介于两者之间的常驻型陪伴产品：
- 比社交宠物更轻，不要求关系链驱动
- 比传统全屏宠物更常驻，核心体验直接发生在桌面
- 比悬浮桌宠更有养成感，具备明确状态、交互和成长阶段

## 目标用户
核心用户默认定义为：
- `18-35 岁`
- 喜欢复古像素、轻陪伴、低负担养成
- 愿意频繁看桌面，但不想被复杂系统和强提醒打扰
- 对“宠物常驻桌面”比“沉浸式长时间游玩”更感兴趣

## 核心价值
V1 需要向用户证明的价值主要有三点：
- 桌面上真的有一个持续陪伴的宠物，而不是一次性摆件
- 用户可以用很低的操作成本维持照顾感和反馈感
- 宠物会随着照顾行为成长，形成轻量但明确的养成回路

## 竞品结论与产品判断
竞品观察结论：
- `Widgetable` 已验证“宠物 + Widget + 情绪陪伴/关系表达”有大市场，核心卖点是共同养宠物和桌面常驻
- `SUSH` 已验证“情侣/双人共养 + Widget”成立，但广告与高黏人感会带来负反馈
- `Mooshi / Shimeji` 更偏悬浮桌宠与屏幕装饰，不是完整养成系统
- `Finch` 与 `Habimon` 说明宠物可承载陪伴与习惯养成需求，但它们不是纯桌面宠物

对应的产品判断：
- V1 不做悬浮宠物，不做双人共养，不做强社交
- V1 先验证桌面常驻、日常回访、基础留存三项能力
- 未来如要做社交、共养、同步，应建立在单人养成模型成立之后

## V1 产品范围
V1 只做以下核心能力：
- 1 只主宠物
- 4 个成长阶段：`孵化 -> 幼体 -> 成长期 -> 成熟体`
- 4 个核心状态值：`饱腹`、`心情`、`清洁`、`精力`
- 4 个核心交互：`喂食`、`玩耍`、`清洁`、`睡觉`
- 2 种桌面组件规格：`2x2` 基础宠物卡、`4x2` 扩展状态卡
- 1 个 App 内详情页，承接宠物状态、背包、成长记录、设置
- 轻提醒能力：饥饿、困倦、脏了、成长完成
- 全程离线可玩，不依赖账号系统

## V1 明确不做
- 悬浮窗宠物
- 实时联机
- 好友、情侣、共养系统
- 复杂房间装修
- PvP、排行榜、公会
- 重剧情驱动
- 多宠物养成系统

## 核心功能清单
### 桌面 Widget
- 展示宠物当前状态与主要反馈
- 允许用户进行低成本、高频次的核心互动
- 在系统重绘或用户点击后快速反映状态变化

### App 内详情页
- 展示更完整的宠物状态信息
- 查看背包和成长记录
- 提供设置入口与后续扩展容器

### 养成系统
- 根据时间推导状态衰减
- 根据互动动作结算状态恢复、成长推进与反馈内容
- 在关键节点触发提醒与成长表现

## 关键用户流程
### 首次体验流程
`首次进入 App -> 领养宠物 -> 引导放置 Widget -> 完成第一次喂食 -> 看到宠物反馈`

### 日常回访流程
`用户看见桌面宠物 -> 发现状态变化或提醒 -> 点击进行喂食/玩耍/清洁/睡觉 -> 宠物即时反馈 -> 用户离开`

### 深度查看流程
`从 Widget 进入详情页 -> 查看状态、成长记录、背包 -> 决定下一步互动 -> 返回桌面`

## 成功指标
V1 重点关注以下三个指标：
- `桌面常驻率`：有多少用户愿意真正把 Widget 放到桌面并保留一段时间
- `日常回访频次`：用户是否形成围绕桌面宠物的轻量交互习惯
- `基础养成留存`：用户是否愿意持续照顾宠物并体验成长阶段变化
