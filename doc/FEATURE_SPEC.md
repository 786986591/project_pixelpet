# project_pixelpet 基础功能清单

## 文档目标
本页定义 V1 基础功能的最小可用版本，重点回答“这个功能在首版到底做到什么程度”。

默认原则：
- 只定义单人、离线、Widget-first 的基础能力
- 优先保证核心养成回路成立
- 不提前引入复杂经济、社交、任务系统

## 功能总览
V1 基础功能建议分为 6 组：
- 宠物创建与初始化
- 宠物状态系统
- 核心互动动作
- Widget 展示与快捷操作
- 轻提醒功能
- 成长记录与基础设置

## 1. 宠物创建与初始化
目标：
- 让用户在首次进入时快速拥有一只可互动的宠物

功能定义：
- 首次进入 App 时自动进入领养流程
- 领养完成后创建 1 只默认主宠物
- 宠物默认从 `孵化` 或初始阶段开始
- 完成创建后立即引导用户放置 Widget

V1 不做：
- 多宠物选择
- 名字自定义
- 初始品类分支

## 2. 宠物状态系统
目标：
- 为养成回路提供稳定的状态变化依据

状态项：
- `饱腹`
- `心情`
- `清洁`
- `精力`

规则要求：
- 四项状态都会随时间推移发生变化
- 用户动作可恢复部分状态
- 状态需要有上下限，避免无限增长或跌破边界
- 状态变化必须能同时影响 Widget 和详情页展示

推荐展示原则：
- 不要求向用户暴露复杂数值计算
- 可以用图标、进度、文案摘要表达状态好坏

## 3. 核心互动动作
### `喂食`
目标：
- 恢复饱腹
- 触发基础正反馈

V1 定义：
- 默认只需支持基础粮食
- 可以预留零食、恢复剂，但不要求首版做复杂消耗策略
- 喂食后宠物要有即时反馈

### `玩耍`
目标：
- 提升心情
- 增强陪伴感

V1 定义：
- 作为轻交互按钮存在
- 可伴随少量精力消耗
- 不做独立小游戏

### `清洁`
目标：
- 恢复清洁状态
- 表达“照顾宠物”的责任感

V1 定义：
- 点击后即时结算
- 可搭配简单气泡或清洁反馈特效

### `睡觉`
目标：
- 恢复精力
- 形成完整照顾闭环

V1 定义：
- 允许通过动作触发休息状态
- 不要求做真实长时睡眠系统
- 可以通过一次动作完成精力恢复和表现切换

## 4. Widget 展示与快捷操作
目标：
- 让用户主要在桌面完成高频回访

V1 定义：
- 至少支持 `2x2` 和 `4x2` 两种 Widget
- Widget 上必须能看到宠物当前状态摘要
- Widget 上至少保留 1 个主 CTA，最好能承载高频互动
- 点击 Widget 后应快速反馈，必要时跳转 App 详情页

形态建议：
- `2x2` 继续作为主宠物卡，强调状态与快捷照顾
- `4x2` 可发展为“鱼缸”主题 Widget，内部承载 `颜文字鱼` 的漂游与互动反馈

`4x2` 鱼缸 Widget 的基础能力建议：
- 至少展示 `1-3` 条颜文字鱼
- 鱼可在鱼缸内部缓慢漂游、转向、停顿
- 点击鱼或鱼缸时有即时反馈，例如波纹、吐泡泡、短暂加速
- 喂食操作可表现为鱼食下落与鱼群聚集
- 若需要更完整操作，再跳转到 App 详情页

交互边界：
- Widget 不承载复杂多步流程
- Widget 不负责完整背包管理和详细记录查看
- 鱼缸互动优先提供情绪反馈和陪伴感，不应演变为复杂小游戏

## 5. 轻提醒功能
目标：
- 在不打扰用户的前提下提醒用户回访

提醒场景：
- 饥饿
- 困倦
- 脏了
- 成长完成

V1 定义：
- 提醒以轻量为主，不做高频轰炸
- 提醒内容应直接指向一个明确状态问题
- 用户从提醒进入后，应能直接完成对应处理

## 6. 成长记录与基础设置
### 成长记录
目标：
- 让用户感知宠物不是静态摆件，而是真的在被养成

V1 定义：
- 记录成长阶段变化
- 记录关键互动摘要
- 展示当前阶段和已达成节点

### 基础设置
目标：
- 让用户控制提醒与基础表现

V1 定义：
- 提醒开关
- 音效开关
- 基础显示偏好

## 基础功能优先级
### P0
- 首次领养
- 宠物状态计算
- 四个核心互动
- Widget 放置后的基础展示与点击反馈
- 详情页状态展示

### P1
- 成长记录
- 轻提醒
- 背包分区
- 简单设置

### 当前不进入 V1
- 多宠物
- 在线同步
- 社交互动
- 家园装修
- 小游戏
- 复杂道具经济系统

## 验收口径
如果以下条件成立，就说明 V1 基础功能基本完整：
- 用户能在首次进入后顺利养到第一只宠物
- 用户主要可在 Widget 上完成高频互动
- 宠物状态会随着时间与操作产生可理解的变化
- 用户可以在 App 内看清状态、成长和基础设置
